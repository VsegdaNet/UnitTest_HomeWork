1) Почему использование тестовых заглушек может быть полезным при написании модульных тестов?

2) Какой тип тестовой заглушки следует использовать, если вам нужно проверить, что метод был вызван
    с определенными аргументами?

3) Какой тип тестовой заглушки следует использовать, если вам просто нужно вернуть определенное значение
    или исключение в ответ на вызов метода?

4) Какой тип тестовой заглушки вы бы использовали для имитации взаимодействия с внешним API или базой данных?

Ответ 1:
 - Тестовые заглушки позволяют изолировать тестируемый модуль от зависимостей
 - Использование заглушек может ускорить выполнение модульных тестов. Из за того что они озволяют
    избежать реальных внешних вызовов
 - Заглушки позволяют контролировать возвращаемые значения и поведение зависимостей.
 - Заглушки позволяют тестировать каждый модуль независимо от других частей системы.

Ответ 2:
 Spy - Шпион в Mockito представляет собой частичную заглушку, которая реально вызывает методы объекта,
 но также записывает информацию о вызовах для последующего анализа в тестах.

Ответ 3:
 Mock - заглушка в Java представляет собой объект,
 созданный для имитации поведения реального объекта в рамках модульного тестирования.

Ответ 4:
 Stub - представляет собой объект, который
 предоставляет заранее определенные ответы на вызовы методов в рамках модульного тестирования.
